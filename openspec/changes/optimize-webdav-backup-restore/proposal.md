# 变更：优化 WebDAV 备份与恢复功能

本次变更旨在优化现有的 WebDAV 备份与恢复功能，解决大文件处理时的内存溢出风险，并增加进度反馈以提升用户体验。

## 动机

当前的备份与恢复实现存在以下问题：
1.  **内存占用高**：备份时将文件完整读入内存，恢复时将整个 Zip 包读入内存，处理大文件或大量数据时极易导致应用崩溃 (OOM)。
2.  **缺乏反馈**：备份与恢复过程在前端无进度显示，用户无法感知操作状态，尤其是当数据量较大时，容易误以为程序卡死。

## 范围

-   **后端优化**：
    -   重构 WebDAV 客户端，支持流式下载与上传。
    -   重构 `backup_webdav` 命令，实现流式下载并直接写入 Zip 流（或分块写入），避免全量加载。
    -   重构 `restore_webdav` 命令，实现流式读取 Zip 并流式上传，避免全量加载。
-   **前端体验**：
    -   在备份与恢复过程中，后端向前端发送进度事件。
    -   前端设置界面增加进度条或状态提示，展示备份/恢复的当前进度（如：正在处理的文件名、百分比）。
-   **健壮性**：
    -   增加对网络中断或错误的更细粒度处理。

## 范围之外

-   自动定时备份。
-   增量备份（仍为全量备份）。
