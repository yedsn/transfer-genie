## ADDED Requirements

### Requirement: 单页产品官网首页
项目 SHALL 提供一个官方单页官网首页，将 Transfer Genie 作为产品进行展示。首页 SHALL 包含 hero 区域、核心价值表达、功能亮点、支持的工作流或架构概览、产品视觉内容，以及指向文档和项目/下载链接的行动入口。

#### Scenario: 访客打开官网首页
- **WHEN** 访客打开官网首页
- **THEN** 访客能够在首屏快速理解 Transfer Genie 的用途
- **AND** 页面提供清晰的后续行动入口，便于继续了解或开始使用

### Requirement: 响应式静态交付
官网首页 SHALL 能以静态资源方式发布，并 SHALL 在桌面浏览器与移动端浏览器中正确呈现。页面的核心内容 MUST 不依赖 Tauri 桌面运行时或服务端应用才能展示。

#### Scenario: 首页在不同设备中打开
- **WHEN** 官网首页在桌面浏览器或移动端浏览器中被打开
- **THEN** 页面布局保持可读且易于导航
- **AND** 核心内容无需依赖应用专属运行时 API 即可访问

### Requirement: README 与官网首页内容一致
官网首页 SHALL 与 README 保持一致的核心定位、主要功能支柱和关键导航目标，使访问者在仓库首页与公开官网之间获得一致的产品叙事。

#### Scenario: 访客在 README 与官网之间切换
- **WHEN** 访客对比仓库 README 与官网首页
- **THEN** 访客看到的是一致的核心产品表达和等价的主要导航入口

### Requirement: 官网预览与发布说明
项目 SHALL 记录维护者如何在本地预览官网首页，以及如何利用生成后的站点文件将其发布到静态托管目标。

#### Scenario: 维护者准备发布官网
- **WHEN** 维护者需要验证或发布官网首页
- **THEN** 仓库中提供清晰的本地预览与静态部署说明
