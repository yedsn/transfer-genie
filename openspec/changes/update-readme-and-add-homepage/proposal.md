## Why

Transfer Genie 已经具备可用的桌面端能力，但当前 `README.md` 更像是功能清单，而不是一个完整、清晰、有吸引力的项目入口。对比 `cc-haha` 这类项目，它还缺少更强的首屏表达、更清楚的信息架构，以及一个面向公开传播的官网首页，来帮助用户快速理解它能做什么、为什么有价值，以及应该如何开始使用。

## What Changes

- 重构根目录 `README.md`，将其升级为更完整的项目门面文档，强化产品定位、章节层次和安装/配置/使用路径。
- 新增一个单页官网首页，让 Transfer Genie 不只是一个代码仓库，而是一个有完整对外展示能力的产品。
- 对齐 README 与官网首页的核心表达，确保产品价值、核心能力、视觉内容、快速开始和文档入口保持一致。
- 优先复用现有应用图标和静态资源，并补充官网首页所需的新增视觉或内容素材。
- 增加可直接落地的静态站点部署说明，便于官网发布。

## Capabilities

### New Capabilities
- `project-docs`: 以仓库首页文档的形式，清晰传达产品价值、核心工作流、快速开始路径、文档导航以及贡献/发布入口。
- `marketing-homepage`: 一个响应式单页官网，延续 README 的核心定位，向访问者提供下载、阅读文档和进一步了解产品的直接入口。

### Modified Capabilities
- 无。

## Impact

- 影响的代码与内容：`README.md`、官网首页源码文件（新的静态站点目录）、共享视觉资源，以及相关文档链接。
- 影响的系统：仓库对外展示、静态站点托管/部署，以及 README 与官网之间的产品叙事一致性。
- 依赖约束：优先采用轻量级静态站点方案或零构建静态资源，除非资源复用确有需要，否则不改动现有 Tauri 应用运行时。
