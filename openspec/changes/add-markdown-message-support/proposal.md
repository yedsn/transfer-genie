# 提案：添加 Markdown 消息支持

本提案旨在增加对 Markdown 格式消息的支持，为用户提供更丰富的文本格式化能力。

## 动机

目前，所有消息均以纯文本形式发送。允许用户发送 Markdown 格式的消息将通过支持标题、列表、粗体/斜体文本和代码块等功能来改善沟通体验。这对于分享代码片段或结构化信息尤为有用。

## 方法

- **消息格式**：在消息结构中添加一个新字段以指定格式（`text` 或 `markdown`）。
- **编辑器**：前端将集成 `editormd` 库。当用户选择 Markdown 格式时，输入框将切换为 `editormd` 编辑器；选择纯文本时，保持原有的文本输入框。
- **渲染**：消息流中的消息将作为 Markdown 渲染。

## 相关能力

- **能力**：`message-feed`
