## 背景
- 新建 Tauri 桌面应用，支持 Windows 与 macOS。
- WebDAV 目录作为共享消息存储。
- 每条消息对应单个文件；本地 SQLite 数据库索引消息元数据。

## 目标 / 非目标
- 目标：发送文本与文件、聊天式列表、可配置发送者名称、可配置刷新间隔、手动刷新、使用 WebDAV 原生认证。
- 非目标：端到端加密、房间/频道、推送通知、分片或断点续传。

## 决策
- 使用配置的 WebDAV 根目录作为单一消息流，不引入房间/线程概念。
- 消息文件名编码元数据，避免依赖服务器自定义属性。
  - 建议格式：`<epoch-ms>__<sender-encoded>__<nonce>__<original-name>`。
  - 文本消息使用 `.txt` UTF-8 内容；文件消息保留原始字节与扩展名。
- 发送者名称以 UTF-8 百分号编码，保证文件名安全；显示时再解码。
- 本地 SQLite 保存消息元数据（id、filename、sender、timestamp、size、kind、etag/mtime、文本内容）。
- 同步循环使用 PROPFIND（Depth: 1），每 N 秒轮询一次；比较文件名 + etag/mtime，拉取新增项。
- 同步时自动下载文件消息到本地存储；不符合消息命名格式的文件忽略且不下载。
- 文本消息列表显示全文内容。
- 手动刷新触发立即同步，并在界面展示刷新状态。

## 风险 / 权衡
- 目录较大时轮询慢：通过缓存 etag/mtime，避免重复下载。
- 文件名编码冲突或非法字符：使用编码后的发送者 + 随机 nonce。
- WebDAV 服务器差异：仅依赖标准 PROPFIND、GET、PUT。

## 迁移计划
- 新应用，无需迁移。
