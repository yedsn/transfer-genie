# 提案：添加图片缩略图与增强预览功能

本提案旨在增强图片文件的处理能力。上传图片时，系统将自动生成缩略图并上传。消息列表将显示这些缩略图。全屏预览功能将升级为支持直接显示图片内容。此外，附件上传预览区域也将支持双击图片进行全屏预览。

## 核心变更

1.  **缩略图生成与存储**: 后端在发送图片文件时，自动生成缩略图，并将其存储在 WebDAV 的 `files/.thumbs/` 目录下。
2.  **消息列表缩略图**: 前端消息列表在渲染图片类型的消息时，会尝试加载并显示缩略图。
3.  **增强的全屏预览**: 点击消息列表中的缩略图或使用“预览”功能时，弹出的全屏预览框将显示图片大图（如果本地已存在或已下载）。
4.  **上传前预览增强**: 在消息发送框下方的附件列表中，双击图片预览图也可以打开全屏预览。

## 技术方案

-   **后端**: 引入 `image` crate 用于图片处理。
-   **后端**: 修改 `send_file` 和 `send_file_data`，在上传原图后，生成缩略图并上传。
-   **后端**: 新增 `get_thumbnail` 命令，用于按需下载并缓存缩略图到本地，供前端展示。
-   **前端**: 更新 `renderMessages` 逻辑，对图片消息请求缩略图。
-   **前端**: 更新 `openMessagePreview` 逻辑，支持 `<img>` 标签展示本地图片。
