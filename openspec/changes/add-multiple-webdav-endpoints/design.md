## 背景
客户端当前仅支持在设置中存储单个 WebDAV 端点。用户希望管理多个端点，支持启用/禁用，并能在首页切换活动端点。

## 目标 / 非目标
- 目标: 支持多个端点、按端点启用状态，以及可更新同步的活动端点切换。
- 非目标: 自动故障切换、按消息路由端点、跨端点消息合并。

## 决策
- 决策: 以列表存储端点，包含稳定 id、URL、认证信息与 `enabled` 标记；`activeEndpointId` 单独保存。
- 决策: 仅允许已启用端点被设置为活动端点；若活动端点被禁用/移除，则清空活动选择并提示用户选择其他端点。
- 决策: 切换活动端点后立即触发同步并刷新消息列表。

## 风险 / 权衡
- 多端点可能导致用户困惑；通过清晰的标签与首页仅展示已启用端点来降低风险。

## 迁移计划
- 升级后首次启动时，将现有单端点迁移到列表中，并设置 `enabled=true` 与 `activeEndpointId`。

## 未决问题
- 已确认：本地消息索引按端点 id 进行隔离。
