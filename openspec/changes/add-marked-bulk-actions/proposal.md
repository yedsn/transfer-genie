# 提案：为标记页增加多选批量操作

## Why
当前标记页只能逐条管理已标记消息。随着标记消息数量增长，用户需要一次性给多条消息补充或调整标签，并且在整理完成后批量删除无用记录，否则整理成本过高。

## What Changes
- 在标记页增加独立的多选模式入口、选择工具栏和全选能力，仅作用于当前标记列表。
- 为标记页增加“批量设置标签”操作，允许用户对当前选中的多条已标记消息统一覆盖标签集合。
- 为标记页增加“批量删除”操作，沿用现有删除范围确认流程，对当前选中的多条标记消息执行删除。
- 保持现有单条标签编辑、单条取消标记、单条删除、标签过滤、搜索和置顶排序行为不变。

## Impact
- Affected specs: `message-feed`
- Affected code: `frontend/index.html`, `frontend/main.js`, `frontend/styles.css`, `src/main.rs`, `src/db.rs`
