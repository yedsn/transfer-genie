# 变更：发送前标记并管理标签

## Why
当前消息只能在发送完成并出现在消息列表后再执行标记。用户如果在发送前就已经明确知道这条消息需要被标记，并希望顺手选择、增删标签，就必须先发送再进入后续管理流程，操作过长。

## What Changes
- 在首页输入区增加独立“标记”勾选入口；鼠标悬停时弹出标签管理层，允许用户为本次发送选择标签、新增标签、删除标签。
- 扩展发送流程，使用户勾选“标记”后，消息在发送成功时直接以已标记状态创建，并带上本次暂存的标签结果。
- 将输入区中的标签新增、删除、选择都视为本次发送前的暂存状态，仅在发送成功后与消息一起保存；发送失败或未发送时不落库。
- 发送成功后重置输入区的标记与标签暂存状态；发送失败时保留当前状态，便于重试。

## Impact
- 影响规格：`message-feed`
- 影响代码：`frontend/index.html`、`frontend/main.js`、`frontend/styles.css`、`src/main.rs`、`src/history.rs`、`src/db.rs`
